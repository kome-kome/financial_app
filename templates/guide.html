Zスコアって何？

このページは、分析タブで使われている手法を数式を使わずに、たとえ話で説明するページです。各分析の画面で「❓ やさしい解説」を押すと、ここの対応する場所へ飛んできます。

もっと正確な数式・前提・参考論文を知りたくなったら、各セクション末尾の「→ もっと詳しく」からモデル解説（技術版）へ進めます。

⚠ どの分析も「未来を当てる魔法」ではありません。判断を助ける参考材料として使ってください。

おすすめ銘柄 ① 銘柄を探す

🟢 ひとことで言うと

通信簿のようなものです。「収益力」「成長」「割安さ」など複数の科目の成績を、あなたが決めた重要度（ウェイト）で合計して、総合点の高い順に会社を並べる仕組みです。理科を重視する子・国語を重視する子で順位が変わるように、ウェイトを変えると上位の顔ぶれも変わります。

📊 何が分かる？

たくさんの会社の中から、自分の好みの条件に合った「良さそうな会社」をまとめて見つけられます。点数の内訳も見えるので、なぜ上位なのかを確認できます。

🛠 どう使う？

  • 「バランス型・成長重視・割安重視・高収益重視」などのプリセットを選ぶ（まずはバランス型でOK）。
  • こだわりがあれば、各指標のスライダー（ウェイト）を動かして重要度を調整する。
  • 実行すると、総合点の高い順に会社が並びます。
各社は指標ごとに「平均からどれくらい良い／悪いか」を表すZスコアに変換してから合計します。これにより、規模や業種の違う会社どうしも公平に比べられます。

⚠ 注意点・落とし穴

ウェイト（重要度）の決め方には数学的な正解はなく、あなたの好みの設定です。「この設定が儲かる」とは限らないので、バックテストで過去の成績を確かめるのがおすすめです。
→ もっと詳しく：モデル解説 6. Zスコア重み付けスコアリング

ネットキャッシュ分析 ① 銘柄を探す

🟢 ひとことで言うと

会社の「財布の中身」を数える分析です。会社が持っている現金など（すぐお金にできる資産）から借金を引いて、手元にいくら残るかを計算します。もし「手元に残るお金 ＞ その会社を丸ごと買う値段（時価総額）」なら、現金の詰まった財布を、財布より安く買えるようなお買い得状態かもしれません。

📊 何が分かる？

会社の値段（時価総額）に対して、手元資金がどれだけ分厚いかが分かります。比率が高いほど「現金的に割安」と見なせます。

比率の目安意味
1.0 以上会社の値段より多くの現金を持っている
0.5 以上値段の半分以上を現金で持っている
0 以下借金のほうが多い（手元はマイナス）

🛠 どう使う？

  • そのまま実行すると、財布の分厚い順に会社が並びます。
  • 「本業でお金を稼げている会社だけ見たい」なら、営業キャッシュフローや純利益がプラスの会社に絞るフィルタをON。
この分析は予測モデルを使わず、決算書の数字をそのまま計算するだけなので、推定のブレが混じりにくいのが長所です。清原達郎『わが投資術』とグレアムの考え方がベースです。

⚠ 注意点・落とし穴

現金が分厚くても万年割安のまま放置される会社もあります。また、商社・銀行・REIT などは決算書の構造が特殊で、この指標がうまく働きません（業種フィルタで外すのがおすすめ）。
→ もっと詳しく：モデル解説 8. ネットキャッシュ分析（清原達郎式）

業種別OLS分析 ② 割安度を測る

🟢 ひとことで言うと

「同じ業種の会社を並べて、相場の値ごろ感を学ぶ」分析です。中古車の値段を「年式・走行距離・グレード」から推測するのと同じで、会社の利益・純資産などの数字から「この会社なら本来このくらいの株価が妥当」という理論株価を計算します。OLSは、その「当てはまりの良い式」を見つける計算方法です。

📊 何が分かる？

業種ごとに「財務の数字 → 妥当な株価」の関係式を作ります。これが次のバリュエーション分析の土台になります。同じ業種の中で比べることで、業種ごとの値ごろ感の違い（IT は高め・銀行は低めなど）に惑わされずに済みます。

🛠 どう使う？

  • 基本はそのまま実行でOK。説明に使う財務項目を増減させることもできます。
  • 「多重共線性の警告」（似た指標を入れすぎたサイン）が出たら、Ridge（リッジ）に切り替えると安定します。
この分析は計算が重いため、本番（Render）環境では実行できないことがあります。その場合は手元のPCで実行すると、結果が共有データベースに保存され本番にも反映されます。

⚠ 注意点・落とし穴

仲間の会社が少なすぎる業種（既定では5社未満）は計算されません。また理論株価はあくまで「業種内の平均的な値ごろ」であり、正解の株価ではありません。
→ もっと詳しく：モデル解説 2. 業種別OLS回帰

バリュエーション分析 ② 割安度を測る

🟢 ひとことで言うと

「理論の値段と、実際の値段のズレ」を測る分析です。業種別OLSが出した「本来このくらい妥当」という理論株価と、市場でついている実際の株価を比べます。実際が理論より安ければ割安（緑）、高ければ割高（赤）。さらに、過去のズレがどのくらいの速さで埋まってきたか（半減期）と、値上がり余地＋配当の「期待総リターン」もまとめて出します。

📊 何が分かる？

  • 乖離率：理論より割安か割高か（プラスが割安）。
  • 期待総リターン：値上がり余地（乖離率）＋配当利回り。「ソート順」で総リターン順に並べ替えられます。
  • implied PER / PBR：理論株価ベースの割安感の倍率。
  • 期待乖離・収束スコア：そのズレが今後どのくらいのペースで縮みそうかの目安。

🛠 どう使う？

  • 先に業種別OLS分析を実行しておく（土台が必要なので、未実行だとこのタブはロックされます）。
  • 実行すると、割安〜割高の順に会社が並びます。「ソート順」で期待総リターン順にも切り替えられます。

⚠ 注意点・落とし穴

「何ヶ月後にどれだけ縮む」「期待総リターン」という数字は、平均回帰を仮定したヒューリスティック（経験則）の参考値であり、当たる確率ではありません。割安に見えても、それには理由（業績悪化など）がある場合もあります。
→ もっと詳しく：モデル解説 3. バリュエーション分析

株価リターン予測 ③ 将来リターンを予測

🟢 ひとことで言うと

「チャートの形＋会社の体力」から、数日〜数ヶ月先の値動きを当てにいく分析です。移動平均からの離れ具合・値動きの荒さ・買われすぎ／売られすぎ（RSI）といったチャートのサインに、ROEやPERといった財務の数字も加えて予測します。

📊 何が分かる？

5日・20日・60日先などの株価リターン（％）の予測と、その予測がどれだけあてになりそうかの目安が分かります。

🛠 どう使う？

  • 予測したい先の日数（例：20日先）を選んで実行。
  • ウォークフォワード検証（＝過去のある時点に立ち、それ以前のデータだけで予測して答え合わせ）の成績で、信頼度を確認します。
「未来のデータをこっそり使ってズルをしない」ように、いつも過去だけで学習→少し先でテストを繰り返して評価します。これがウォークフォワードCVです。

⚠ 注意点・落とし穴

直線的な関係しか捉えられないため、複雑なパターンは苦手です。また財務データは年1回しか更新されないため、短期予測では同じ財務値が使い回されます。短期の値動き予測は特に外れやすい点に注意。
→ もっと詳しく：モデル解説 4. 株価リターン予測（月次WF-CV）

マクロ×リスク-リターン推奨 ③ 将来リターンを予測

🟢 ひとことで言うと

「もうけの大きさ」だけでなく「どれだけブレるか（リスク）」も一緒に見て、コスパの良い会社を探す分析です。さらに、為替や米国株などの世の中の流れ（マクロ）も取り込み、「円安に強い会社／弱い会社」といった相性まで考えます。横軸リスク・縦軸リターンの地図の上で、同じリスクならより高リターンな会社（効率的フロンティア）を浮かび上がらせます。

📊 何が分かる？

リスクとリターンのバランスが良い会社が、散布図（バブルチャート）と一覧で分かります。どの要因（財務・マクロ）が効いているかの目安も棒グラフで見られます。

🛠 どう使う？

  • 使いたい財務要因・マクロ要因を選んで実行（既定のままでもOK）。
  • リスク回避度（λ）スライダーや横軸のリスクの種類を動かすと、再計算なしですぐに並びが変わります。安全志向なら λ を大きめに。
「リスク」には、予測の不確実さ・過去の値動きの荒さ・モデルの当てはまりの悪さなど数種類があり、好みで横軸を切り替えられます。

⚠ 注意点・落とし穴

最も高度な分析で、当てはまりの精度（CVのR²）は高くありません。表示される要因の効き具合は「向き（株高/株安）と大小の目安」として読み、効果を過大に信じないでください。
→ もっと詳しく：モデル解説 9. マクロ×リスク-リターン推奨

バックテスト ④ 戦略を検証

🟢 ひとことで言うと

「もし過去にこの作戦で選んでいたら、儲かった？」を確かめる答え合わせです。Nヶ月前の時点で分かっていた情報だけを使って会社を選び、その後の実際の株価でどうなったかを計算します。

📊 何が分かる？

スコアリングで選んだ上位の会社が、その後どれだけ上がった／下がったか。さらに「全体平均（ベンチマーク）」と比べて勝てていたか（超過収益）が分かります。

🛠 どう使う？

  • 何ヶ月前から検証するか・上位何社を見るかを選んで実行。
  • 上位グループの平均リターンが、全体平均を上回っているかを確認します。

⚠ 注意点・落とし穴

過去がよくても未来を保証しません。さらに、今もうデータベースに残っている会社だけを対象にするため（＝途中で消えた会社が含まれない＝生存バイアス）、結果は実際より良く見えがちです。
→ もっと詳しく：モデル解説 7. バックテスト

売り候補ランキング ⑤ 保有を見直す

🟢 ひとことで言うと

「おすすめ銘柄」がこれから買う候補を探すのに対し、これはすでに持っている銘柄の中から「そろそろ売った方がよさそうな順」に並べる仕組みです。買いの裏返しで、割高・業績が悪化・株価が下げ始めの3つの観点から「売り点数」をつけ、各銘柄に売却 / 一部売却 / 保有継続の札を貼ります。

📊 何が分かる？

持ち株を一覧で並べ、どれが相対的に手放す候補かが分かります。さらに「今が売り時か（株価のトレンド）」も加味して、割高でも上昇中の銘柄はあわてて売らないよう調整します。取得単価を入れれば損益（含み益・含み損）も表示します。

🛠 どう使う？

  • 持っている銘柄を1行ずつ入力します（証券コードだけでもOK。例: 7203, @1500, 2024-03-10）。入力はこのブラウザに自動保存されます。
  • 「割高警戒型・業績悪化重視・バランス型」などのプリセットを選ぶか、スライダーで観点の重みを調整します。
  • 実行すると、売り点数の高い順に並び、売却 / 一部売却 / 保有継続の札が付きます。
各観点は同じ年の全社の中で「平均よりどれくらい良い／悪いか」（Zスコア）に直してから合計するので、観点の単位の違いに引きずられません。「割高度」を使うには先に業種別OLS分析を実行してください。

⚠ 注意点・落とし穴

これは「いつ売れば儲かる」を当てるものではなく、持ち株を見直すきっかけです。全部が優良なら全銘柄「保有継続」になります。ETF や外国株など未収録の銘柄は判定できません。
→ もっと詳しく：モデル解説 10. 売り候補ランキング

Zスコアって何？ 共通の考え方

🟢 ひとことで言うと

テストの「偏差値」とそっくりな考え方です。ある年の全社の中で、その会社の数字が平均よりどれくらい上か下かを、ものさしを揃えて表した点数です。0が平均、プラスなら平均以上、マイナスなら平均以下。売上のような「桁の大きい数字」も、利益率のような「％の数字」も、同じものさしに乗せて比べられるようになります。

📊 なぜ使うの？

会社ごとに規模も単位もバラバラな指標を、そのまま足し算すると桁の大きい指標だけが効いてしまいます。Zスコアに直しておくと、どの指標も対等に比べられるので、おすすめ銘柄のスコアリングなどで公平な合計ができます。

同じ年どうしで比べるのがポイントです。好景気の年と不景気の年を混ぜると比較が歪むため、年ごとに平均と散らばりを計算しています。
まぎらわしい注意：ここでの「Zスコア」は偏差値の仲間（標準化スコア）で、倒産予測で有名な「Altman の Z スコア」とは別物です。
→ もっと詳しく：モデル解説 5. 横断的Zスコア正規化
