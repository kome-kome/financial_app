共通事項 全モデル適用

外れ値処理（全モデル必須）

winsorize(X, lo=p1, hi=p99) → X を [第1百分位, 第99百分位] の範囲にクリップ

日本株データは BPS・EPS に p99 の数百倍の外れ値が存在し、無処理では OLS の行列反転が数値的に破綻する（R² が −10³² になる等）。実装: plugins/utils.py の winsorize()

次元整合性の原則

OLS 係数が経済的に解釈できるためには、説明変数と被説明変数の次元が一致していなければならない。

✅ 正しい例❌ 誤った例
EPS/BPS [円/株] → 株価 [円/株]ROE [%] → 時価総額 [百万円]
ログリターン [無次元] → ログリターン [無次元]自己資本比率 [%] → 株価 [円]

OLS実装（numpy + scipy + statsmodels）

β = (X'X)⁻¹ X'y # 正規方程式 最小二乗法: numpy.linalg.lstsq（SVD ベース、数値安定） p値計算: scipy.stats.t.sf（df < 30 でも正確） 詳細診断: statsmodels.OLS（Durbin-Watson・Jarque-Bera・F検定・HC3 SE） 返り値: beta, yhat, r2, adj_r2, rmse, mae, se, t_stat, p_value, df, rank, condition_number

係数の有意性（標準誤差・t統計量・p値）

σ² = SSE / (n - p) # 残差分散 Var(βᵢ) = σ² × [(X'X)⁻¹]ᵢᵢ # 係数の分散 SE(βᵢ) = √Var(βᵢ) # 標準誤差 tᵢ = βᵢ / SE(βᵢ) # t統計量 pᵢ = 2 × (1 - Φ(|tᵢ|)) # 両側p値（df ≥ 30）

df = n − p。p 値は scipy.stats.t.sf による正確な計算で、df < 30 の小サンプルでも信頼できる。慣例的に p < 0.05 を有意とみなす。

詳細統計診断（statsmodels）

Durbin-Watson: 残差自己相関の検定（1.5〜2.5 なら問題なし） Jarque-Bera: 残差正規性（歪度・尖度） F統計量・p値: モデル全体の有意性検定 HC3 SE: 不均一分散に対応したロバスト標準誤差

業種別 OLS の結果に diagnostics フィールドとして含まれる。

多重共線性チェック（VIF・Pearson相関）

VIFᵢ = 1 / (1 - R²ᵢ) # R²ᵢ は特徴量 i を残りで回帰したときの決定係数

慣例: VIF > 10 または |Pearson r| > 0.9 で多重共線性ありと判断。業種別OLS結果に collinearity_warnings として併記される。

Ridge 回帰（L2 正則化）

βridge = arg min ‖Xβ - y‖² + α‖β‖² # 最適 α は CV で自動選択

業種別 OLS の regularization="ridge" パラメータで切替可能。多重共線性が顕著な業種では予測安定性が向上するが、SE / t / p 値は定義されない（NaN）。実装は sklearn.linear_model.RidgeCV。

分析の3層モデル 設計の枠組み（ADR-0001）

分析手法は目的の異なる3つの層で整理している（CONTEXT.md「分析の階層」）。層が違えば導出の種類が違うため、フラットな一覧ではなく層で束ねて把握する。本ページの各モデルは下表の層・目的別カテゴリに属する。

層入力 → 出力該当カテゴリ
一次分析個別銘柄 → 銘柄ごとのシグナル（スコア／割安度／リターン予測）。利用者が直接アクションする。① 銘柄を探す ／ ② 割安度を測る ／ ③ 将来リターンを予測

Zスコア重み付けスコアリング おすすめ銘柄

plugins/recommend.py

各指標の Zスコアを重み付け線形結合してスコアを計算し、企業をランキングする。

スコア計算式

score_i = Σⱼ∈present (weight_j × z_metric_j_i) / Σⱼ∈present |weight_j| present : 企業 i において値が NULL でない指標の集合 weight_j : 指標 j の重み（ユーザー設定） z_metric_j_i: 企業 i の指標 j の Zスコア（年度別正規化済み）

weighted mean で計算するため、指標カバレッジが異なる銘柄を公平に比較できる。

カバレッジフィルタ

coverage_i = Σⱼ∈present |weight_j| / Σⱼ |weight_j|

min_coverage（デフォルト 0.5）未満の企業はランキングから除外する。1.0 指定で全指標が揃った企業のみ対象。

バランス型1.01.00.80.80.50.5
成長重視1.00.52.00.5—0.3
割安重視1.01.0——0.52.0
高収益重視2.02.0—1.00.5—

gap_ratio は業種別OLS乖離率であり、正値 = 割安の指標として機能する。

限界: ウェイト設定に数学的根拠はなく直感的なヒューリスティック。モデルの有効性はバックテストで検証すること。

ネットキャッシュ分析 清原達郎式

plugins/net_cash_analysis.py

清原達郎『わが投資術』(2024) で提唱された ネットキャッシュ および ネットキャッシュ比率 で割安株をスクリーニングする。OLS・回帰モデルを介さず会計値からの直接計算であるため、推定誤差が混入せず堅牢である点が特徴。あわせて、より保守的な グレアムの NCAV（純流動資産価値）・NCAV比率 を併設し、2 系統で割安銘柄を相互検証できる。

数式

# ネットキャッシュ（円・清原式） NC = 流動資産 + 投資有価証券 × 0.7 − 総負債 nc_ratio = NC / 時価総額 # NCAV（円・グレアム1934、0.7補正なしのより保守的な原型） NCAV = 流動資産 − 総負債 ncav_ratio = NCAV / 時価総額 # 実装上の単位調整（market_cap は百万円単位） 比率 = 金額 / (market_cap × 1_000_000)

銘柄選別基準

比率意味
nc_ratio ≥ 1.0現金で買える 時価総額より多くのネットキャッシュを保有
nc_ratio ≥ 0.5半額バーゲン 時価総額の半分以上をネットキャッシュで保有
nc_ratio ≤ 0負債超過 ネットキャッシュがマイナス
ncav_ratio ≥ 1.5グレアム ネットネット 時価総額 < NCAV × 2/3（グレアムの 2/3 ルール）

フィルタ設計（データ品質ガード・バリュートラップ除外）

① データ品質ガード（NC比率の上限・既定ON）: market_cap は実測ではなく推計株数（純資産÷BPS）ベースの概算で、推計が壊れると時価総額がほぼ 0 になり比率が異常値（数十〜数万倍）に発散する。nc_ratio > SANITY_MAX_NC_RATIO（既定 5.0、空/0で無効）を除外し、ランキング上位が推計崩れデータで埋まるのを防ぐ。割安基準ではなく純粋なデータ品質ガードであり、かつての一律時価総額下限（小型バーゲンまで巻き込む鈍器）を置かずに異常値だけをピンポイント除去できる。最低時価総額は任意フィルタに降格（既定無効）。

② バリュートラップ除外（任意・既定OFF）: 割安でも現金を毀損し続ける企業は万年割安の罠になりやすい。require_positive_ocf（営業CF>0）・require_positive_ni（純利益>0）を任意で要求できる。データ欠損（NULL）は判定不能として通す。

投資有価証券に 0.7 を乗じる理由

清原氏の経験則。投資有価証券は (1) 時価評価のブレで簿価との乖離が大きく、(2) 売却時に含み益課税（法人実効税率 ≈ 30%）が発生するため、保守的に 簿価の 70% でカウントする。実装定数 INVESTMENT_DISCOUNT = 0.7。

計算フロー

# 1. 財務収集時（calc_derived） net_cash = current_assets + investment_securities × 0.7 − total_liabilities → FinancialRecord.net_cash カラムに保存 # 2. 市場データ更新時（update_market_data_only） nc_ratio = net_cash / (market_cap × 1_000_000) → FinancialRecord.nc_ratio カラムに保存 # 3. Zスコア正規化（_calc_zscore_for_year） z_nc_ratio = (nc_ratio − μ_year) / σ_year → FinancialRecord.z_nc_ratio カラムに保存

投資有価証券の XBRL マッピング

会計基準XBRL 要素
JGAAPInvestmentSecurities / InvestmentsInSecurities / ShortTermInvestmentSecurities
IFRSOtherFinancialAssetsNonCurrentIFRS（近似値）

IFRS には完全に対応する科目がないため「非流動その他金融資産」で近似する。古いレコード（本機能リリース前）は bs_investment_securities = NULL で、内部計算では 0 として扱う（簡易 NCAV (Graham 1934) 相当の値を返す）。

限界:
• 0.7 倍ルールは経験則で、個別銘柄の含み益・含み損・政策保有株は反映しない
• 特別損失リスク: 売掛金・棚卸資産が現金化できる保証はない
• IFRS 採用企業では OtherFinancialAssets の区分が異なる場合がある
• 商社・金融・REIT 等は BS 構造が特殊で指標が機能しない業種がある（業種フィルタで除外推奨）
• 古いレコードは投資有価証券未収集（NULL=0扱い）のため過小評価方向のバイアスあり
• 時価総額の推計誤差: 発行株式数を 純資産÷BPS で近似するため、推計が崩れると market_cap が過小になり比率が上振れする。サニティ上限で異常値は除くが、上限以下でも比率はやや楽観方向のバイアスを含む（NCAV比率も同じ分母のため同様）

業種別OLS回帰

plugins/sector_ols.py

業種ごとに独立して OLS 回帰を実行し、Ohlson モデル拡張型で株価（円/株）を推定する。業種内でモデルを構築することで、業種間の P/E・P/B 構造差の影響を排除する。

次元整合性の構造的強制（CLAUDE.md 制約）: 目的変数は stock_price [円/株] 単一に固定、説明変数は per-share [円/株] のみ。 UI/API レベルで他の組み合わせを選べないようにすることで、係数 β を「implied 倍率」として経済的に解釈可能な状態に保つ。

モデル定式化

# 業種 s 内で独立に実行（Ohlson 拡張型） ŷₛ_norm = β₀ + Σⱼ βⱼ · Xₛⱼ_norm y = stock_price [円/株] Xⱼ = per-share 財務金額 [円/株] # 派生 per-share キー (ps_*) はランタイム計算 shares = bs_total_equity / bs_bps [株] ps_<feat> = <絶対額カラム> / shares [円/株] # 業種内でのwinsorize + z-score正規化 Xₛⱼ_norm = (Xₛⱼ_winsorized − μₛⱼ) / σₛⱼ # 乖離率（gap_ratio） gap_ratio = (ŷ − y_actual) / y_actual × 100 [%] gap_ratio > 0: 割安（市場が過小評価） gap_ratio < 0: 割高（市場が過大評価） # 予測値の DB 書き込み（互換のため百万円へ換算） predicted_market_cap = ŷ_pred / stock_price × market_cap [百万円]

説明変数（全項目 [円/株] per-share）

説明変数のキーは DB永続 per-share（pl_eps, bs_bps, dps — 公式開示値）と 派生 per-share（ps_* — 絶対額カラムを発行株数で割って実行時計算）の 2 系統。

デフォルト10項目（PL/BS/CF を網羅）:

カテゴリキー説明
PL（公式）pl_epsEPS 公式値
BS（公式）bs_bpsBPS 公式値 — Ohlson モデル中核
還元（公式）dps1株配当 公式値
PL（派生）ps_revenue売上トップライン

派生 per-share の全選択肢は PL/BS/CF の絶対額カラムを網羅（約37項目、ps_* プレフィックス）。うち C2 収集列（研究開発費 ps_rd_expenses・減価償却費 ps_depreciation・有形固定資産合計 ps_ppe_total・投資その他の資産合計 ps_investments_other_assets・特別損益 ps_extraordinary_income/ps_extraordinary_loss）も per-share 派生として選択可能。ただし欠損が広い（特別損益は JGAAP 専用・研究開発費は非研究開発企業で null）ためデフォルト10項目には含めず、研究開発集約度・資本集約度を見たい業種で明示選択する運用。

実行条件: 業種最小サンプル数（デフォルト: 5社）を満たさない業種はスキップ。各銘柄に発行株数が必要で、issued_shares（XBRL 直接値・fill率100%）を優先し、欠損時のみ bs_total_equity ÷ bs_bps で推計する。どちらでも求められない銘柄のみ対象外（bs_bps 欠損だけでは除外されない）。

注意:
  • 発行株数を issued_shares・bs_total_equity ÷ bs_bps のいずれでも推計できない銘柄のみ集計対象から除外
  • 説明変数の自動ドロップ: 選択列のうち欠損率が 50% を超える列、および AND 除外でどの業種も最小サンプル数に届かない場合の欠損最多列を自動除外し、除外内容を画面に警告表示（欠損列を重ねて 0 業種に潰れる事故を防ぐ）
  • per-share 10項目以上選択時は PL同士・BS同士の比例関係から VIF>10 が頻発する。check_collinearity の警告が出た業種では Ridge への切替を強く推奨
  • 株数推計は IFRS/JGAAP 定義差・期中増資・優先株・転換社債存在時に誤差あり（FUTURE_TASKS の J-Quants IssuedShares 取得で根本解決予定）
  • バリュエーション分析（gap_analysis）は本プラグインの実行後でなければ利用不可。gap_ratio の収束予測には統計的根拠がない（→「バリュエーション分析」参照）

バリュエーション分析 割安度＋平均回帰＋期待総リターン

plugins/gap_analysis.py（内部 slug・/api/gap-analysis は後方互換で維持）

業種別OLS（②割安度を測る）が推定した理論値と実際値の乖離率（割安度）を起点に、平均回帰タイミング（半減期）と期待総リターン（乖離＋配当利回り）を一括で出すバリュエーション系のハブ。各銘柄の年次 gap_ratio 履歴に対して statsmodels の AR(1) モデルを MLE 推定して半減期を計算する（履歴 8 観測未満はヒューリスティックにフォールバック）。旧「総合リターン予測」プラグインの独自 OLS は廃止し、本モデルへ統合した（ADR-0001）。

期待総リターンと implied 倍率（旧 total_return 吸収）

# gap_ratio[%] と配当利回り[%] は同次元 → 加算可能 予測株価 = 実株価 × (1 + gap_ratio / 100) [円/株] 期待総リターン = gap_ratio + 配当利回り [%] # sort=total_return で降順表示 implied PER = 予測株価 ÷ EPS, implied PBR = 予測株価 ÷ BPS

配当利回りの異常値（30%超）は 0 とみなす。min_div_yield（%）で最低配当利回りフィルタ。プール回帰の β 係数（市場全体の implied 倍率）は失うが、業種別係数（業種別OLS）で代替する。

AR(1) MLE による半減期推定（推奨パス）

xt = c + φ xt-1 + εt, εt ~ N(0, σ²) # 平均回帰条件 0 < φ < 1 # 半減期 half_life = −ln(2) / ln(φ) [年]

フォールバック: ヒューリスティック（履歴不足時）

# 履歴 < 8 観測または AR(1) 推定失敗時のみ使用 half_life = max(6, min(24, |gap| / 2)) [ヶ月]

共通: 期待乖離率と収束スコア

# n ヶ月後の期待乖離率（指数減衰） gap_t = gap₀ × exp(−ln(2) / half_life × t) # 収束スコア（参考値、0–100スケール） conv_score₁₂ₘ = max(5, min(95, 50 + gap₀ × 0.8))

③ 将来リターンを予測 一次分析・リターン予測

将来リターンそのものを推定する。ウォークフォワードCVで自己検証を内蔵する。

マクロ×リスク-リターン推奨

財務指標とマクロ要因の交差項 OLS で1年先リターンを推定し、リスク-リターン平面の効率的フロンティア（Pareto 最優解）を提示するモデル。

説明変数

被説明変数 μ（1年先＝52週先週次ログリターン年率）、全変数は無次元。OLS 学習前に全特徴量へ winsorize(p1–p99)→z-score 標準化を適用。財務特徴量・マクロ特徴量は分析画面の M-1 タブで選択できる（fin_features / macro_features multiselect）。

  • 財務（価格由来＝バリュー）: PER / PBR（既定）
  • 財務（価格フリー）: ROE / ROA / 自己資本比率 / D/E / 営業CF/売上 / EPS成長率 / 営業利益成長率 / R&D集約度 / D&A集約度 / Z スコア3種（FinancialMetric VIEW）。既定は ROE・ROA・自己資本比率・EPS成長率
  • モメンタム: 12-1ヶ月ログリターン — Jegadeesh-Titman (1993) アノマリー（use_macro 時）
  • マクロ (innovation): 市場系 USDJPY / SP500 / NIKKEI225 / TOPIX = YoY 変化率、US10Y 等 = 5年 Z スコア、＋日本実体経済（実質GDP=YoY、失業率・貿易収支=Zスコア。FRED 経由）＋鉱工業指数（生産・在庫=YoY。e-Stat 経由・FRED 版が2024年に配信停止したため切替）、いずれも公表ラグ補正済み — Chen-Roll-Ross (1986) 型（既定は米国寄り USDJPY・SP500・US10Y を維持。TOPIX は ETF 1306.T で収集修正済。日次の日本10年金利は ^JGB 廃止で非対応）
  • 交差項: 選択財務 × 選択マクロ + セクターダミー × マクロ（APT の異質感応度 β を捕捉）

PER/PBR は循環参照ではない：目的変数は株価水準でなく将来リターンのため、現在の PER/PBR で将来リターンを予測するのは正統なバリュー・ファクター（Fama-French HML ≒ 1/PBR）。価格を含まないファンダ（ROA・EPS成長率 等）を既定に併置し「割安」と「価格の平均回帰」を分離できるようにしている。

特徴量選択（LASSO-LARS / BIC）

BIC = n · log(SSE / n) + k · log(n)

LassoLarsIC(criterion="bic") で LARS パスを1パス計算し BIC 最小点を選ぶ（標準化後）。L1 正則化が共線性をネイティブに処理するため VIF 門番は不要。BIC 最小解が上限を超える場合は |係数| 降順の上位 max_features に切り詰め、選択済み特徴量で最終 OLS を再フィットして不偏化する（LASSO は選択専用）。

効用関数と Pareto フロンティア（クライアント側即時）

U = μraw − λ · Raxis

λ はリスク回避度スライダー（0〜5、既定 1.0）。Raxis は risk_axis で選んだリスク（R2 既定 / R_macro）。期待リターン基準は μraw（OLS の生予測値）。
効用 U・Pareto 判定・並べ替え・top_n・R3 ゲートはモデル再学習に関与しない後処理のため、サーバーは全社の raw 値（mu_raw / r1 / r2 / r3 / r_macro）を返し、これらはブラウザ側で算出する。よって λ 調整・リスク軸切替・表示件数・R3 ゲートは再計算なしで即時反映される（重い計算は特徴量・マクロ等モデル本体を変えた時のみ）。
バブルチャートのマッピングは y=μraw / x=選択リスク軸（R2/R_macro・type:linear）/ 色の濃淡=効用 U / 枠線強調＋フロンティア線=Pareto 優位。散布図は R3 ゲート後の全社を描画し効用上位 top_n を強調、両軸は [p1,p99] にクランプ。Pareto 判定軸は risk_axis に追従する。

選択特徴量の係数可視化：execute は最終 OLS の標準化係数を feature_coefs で返し、UI は |β| 降順の横バーで表示する（財務／マクロ／交差項／テクニカルを色分け、ゼロ中心で符号＝株高/株安方向）。CV R² は低いため、係数は符号と相対的大小の目安であり寄与度の過大解釈は避ける。

M-2: マクロ×財務 勾配ブースティング 将来リターンを予測

M-1（マクロ×リスク-リターン推奨）の非線形兄弟。同一のデータと目的変数（52週先対数リターン）を使い、勾配ブースティング決定木（XGBoost）が金利レジーム×バリュー・ボラ局面×収益性などの非線形・高次の交互作用を自動学習する。

M-1 との比較が価値の核心。同一 walk-forward フォールドで XGBoost と素 OLS の mean R²/RMSE を side-by-side で出力し、どちらが有効かをユーザー自身が判断・改善し続けられる。

主な違い（M-1 と比較）

交差項 なし（木が交互作用を自動学習）
特徴量選択 BIC/LASSO なし（全特徴量を投入し木の暗黙選択）
正則化 max_depth=4・min_child_weight=5 等の強正則化デフォルト（低 S/N 対策）
R1 なし（OLS 固有の予測 SE。効用軸ではないため欠落しても幾何は壊れない）
SHAP グローバル mean|SHAP|（大きさのみ・方向なし）＋全社の per-stock 寄与内訳
X winsorize 撤去（木は単調変換不変のためクリップ不要）、y winsorize は維持

SHAP による解釈

SHAP（SHapley Additive exPlanations）は各特徴量が予測値にどれだけ寄与するかを理論的に一意に分解する手法。M-1 の標準化係数バーに相当する位置に global mean|SHAP| を表示し、ランキング行をクリックするとその銘柄がなぜ推奨されるかを特徴量別に確認できる。

注意：mean|SHAP| は大きさのみ（非負）。M-1 の係数（正＝リターン↑・負＝リターン↓）と異なり、符号情報は per-stock SHAP パネルで確認する。

アウトオブサンプル検証（OOF）と売り推奨連携

M-2 は walk-forward CV の無リーク OOF 予測だけを使い、再学習なしで「予測 μ̂ が将来リターンを順序付けるか」を検証する（既存「バックテスト」＝preset/as-of のポートフォリオ模擬とは別物）。各期で μ̂ を横断分位し、分位別の実現リターン・rank-IC（順位相関）・ロングショート spread・hit-rate を出力する。μ̂ 水準の時系列ドリフトに頑健な期内横断（per-period）分位を採る。

あわせて各銘柄の μ̂ を保存し、売り候補ランキングが「μ の出所」トグルで M-1／M-2 を選べる（既定は M-1）。M-2 を一度ローカルで実行すると、以降の売り判定で M-2 の μ が使える（未実行なら自動でその成分を除外）。M-2 は予測 SE を持たないため、売り側の R3 足切りゲートは M-2 選択時は無効。設計の記録は ADR-0004。

M-3: ベイズ状態空間モデル（時変マクロβ DLM） 将来リターンを予測

M-1（OLS 線形）・M-2（XGBoost 非線形）の第3の兄弟。両者が「ある時点で係数を1つ推定する（静的係数）」のに対し、M-3 は係数そのものが時間とともに変動する動的線形モデル（DLM）を、銘柄ごとに独立なカルマン型逐次ベイズ更新で推定する。マクロ感応度 β の推移と期待リターン α の現在水準を信用区間つきで提示する。

モデル定式化

各銘柄を独立に、週 t で次の状態空間モデルとして推定する：

観測: y_t = F_t′ θ_t + ν_t # y = 週次対数リターン F_t = [1, Δm_1, …, Δm_k]′ # 定数項 ＋ マクロ週次変化 θ_t = [α_t, β_1, …, β_k]′ # 時変アルファ ＋ 時変マクロβ システム: θ_t = θ_{t−1} + ω_t # ローカルレベル＝ランダムウォーク

主な特徴

構造 銘柄別 TVP 時系列（per-stock）。銘柄固有の時変マクロ感応度を得る
観測 週次リターン × マクロ週次変化（同時点ファクタ応答 / APT 型）
ファクター 既定 USD/JPY・米10年金利・日経225・WTI（市場ファクターを含め α は固有アルファ）。TOPIX（広範市場・1306.T 連動）を選択肢に追加。日次の日本10年金利は ^JGB 廃止のため月次 FRED 据置
推定 自前 割引係数 DLM（West & Harrison 型）。割引 δ で状態ノイズを表現し銘柄ごとの最適化が不要
不確実性 観測分散を Normal-Gamma 共役で学習。予測分布は Student-t で α/β の信用区間を解析的に算出
µ̂ 最新フィルタ αT を年率化（αT × 52）。M-1/M-2 と同じ µ̂ ランキング

検証（1期先予測診断）

DLM は各週 yt を観測前に予測する。標準化予測誤差（バーンイン除外）から、校正（標準化誤差²の平均・1 が理想）・予測 RMSE・95% 信用区間カバレッジを全銘柄平均で集計する。状態空間モデルに固有の自己診断で、M-1/M-2 の walk-forward CV とは別枠。

限界

β はマクロ「ショックへの感応度」であり予測子ではない（将来のマクロ変化は未知）。µ̂ はマクロで説明されない持続的ドリフト α に由来する。週次・最低約2年の履歴を要する。初版は API/UI のみで、売り候補ランキング連携は将来対応。

④ 戦略を検証 メタ検証

一次分析の実績有効性を過去データで事後検証する（分析の分析）。scoring source で各手法を同一土俵に乗せる。

バックテスト

backtest.py — GET /api/backtest（routers/analysis.py）

過去 N ヶ月前の時点で確定していた財務データを使いスコアリングし、その後の実際の株価リターンを計算することでモデルの有効性を事後的に検証する。検証対象のスコアリング手法は source で切替（recommend ＝複合スコア／valuation ＝期待総リターン gap_ratio＋配当／net_cash ＝清原式ネットキャッシュ比率／sell ＝売り候補＝買い系スコアの符号反転）。ランキングを出す一次分析を同一土俵で比較できる（メタ層の一般化）。sell は超過収益が負（上位＝売り候補が下回る）ほど有効と読む（メタ層×双対層）。

計算ロジック

start_date = today − months_ago × 30日 1. start_date 以前に period_end が確定しているレコードで 各社の最新年度データを取得 2. source のスコア関数で全社をランキング （recommend=加重和 / valuation=gap+配当 / net_cash=NC比率） 3. 上位 top_n 社について: 始値 = start_date 以降の最初の終値 終値 = 最新の終値 4. 実績リターン = (終値 − 始値) / 始値 × 100 [%] # ベンチマーク benchmark = スコアリング対象企業（最大500社）の平均リターン 超過収益 = 上位N社平均リターン − ベンチマーク平均
バイアスに注意:
• 生存バイアス: 現在 DB に存在する企業のみ対象（過去に上場廃止した企業は含まれない）
• 決算ラグ: period_end ≤ start_date の制約のみで、実際の報告書公開日（+45–60日）は未考慮
• ベンチマーク: 全上場銘柄ではなくDBに収録された企業の部分集合

⑤ 保有を見直す 双対分析

一次分析を逆観点で保有銘柄へ向け直し、売るべき銘柄と売り時を出す。

売り候補ランキング 保有銘柄の売り時

plugins/sell_ranking.py

買い系の一次分析（おすすめ銘柄・業種別OLS／バリュエーション分析）が全銘柄ユニバースから「買い」を探すのに対し、本モデルはユーザーが入力した保有銘柄リストの中から「売るべき銘柄と売り時」をランキングする。観点は買いの「逆」：① 割高度（回帰乖離 gap_ratio が負＝割高）、② 業績悪化（ROE・営業利益率・CF・成長率の低さ）、③ 価格モメンタム（下落トレンド＝売り時）。保有はサーバ非保存（都度入力＋ブラウザ記憶）、購入単価は損益表示のみ。

スコア計算式（スケール整合）

％指標（gap_ratio・rev_growth）と無次元比率（roe 等）が混在するため、次元整合性の原則（共通事項）に従い最新年度ユニバース全体で winsorize → z 標準化してから合成する。

sell_score_i = Σⱼ∈present (weight_j × (−z_metric_j_i)) / Σⱼ∈present weight_j weight_j : 観点 j を売り判断で重視する度合い（非負） z_metric_j_i : 企業 i の指標 j のユニバース標準化値（±5 クリップ）

符号反転 −z により、平均より劣る（割高・低収益・低成長）銘柄ほどスコアが正に大きくなる。平均並みは ≈0。カバレッジ下限を下回る銘柄は「データ不足」。

価格モメンタム（タイミング軸）

stock_price_weekly から 13週リターン・52週高値からの下落を算出し、トレンドを 下落（13週 ≤ −10%）/ 上昇（≥ +10%）/ 横ばい に分類する。

アクションラベル

絶対閾値: スコア ≥ sell_threshold（既定 0.8）→ SELL、≥ reduce_threshold（既定 0.3）→ REDUCE、未満 → HOLD。タイミング補正（既定 ON）: 下落は 1 段引き上げ、上昇は SELL→REDUCE へ緩和（上昇中の即売り回避）。相対ランキング＋絶対ラベルの併用により、優良な保有のみなら全 HOLD になる。

限界: 割高度には業種別OLS（②割安度を測る）の事前実行が必要（未実行なら 400）。DB 未収録（ETF・外国株）は判定対象外。ウェイトは直感的ヒューリスティックで、投資助言ではなく保有整理の参考スコア。

財務指標を年度別に横断的な Zスコアに変換し、異なる企業間で指標の相対的な優劣を比較可能にする。

Altman の Z スコアとは別概念。 ここでの「Zスコア」は標準化スコア（standard score）であり、倒産予測モデルである Altman (1968) の Z スコアとは無関係。

年度別計算の理由

異なるマクロ環境（金融緩和期・引き締め期等）の年度を混在させると比較が無意味になる。年度をまたいで計算すると、例えば低金利期の高 PER 水準が正常値として紛れ込む。
