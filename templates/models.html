共通事項 全モデル適用

外れ値処理（全モデル必須）

winsorize(X, lo=p1, hi=p99) → X を [第1百分位, 第99百分位] の範囲にクリップ

日本株データは BPS・EPS に p99 の数百倍の外れ値が存在し、無処理では OLS の行列反転が数値的に破綻する（R² が −10³² になる等）。実装: plugins/utils.py の winsorize()

次元整合性の原則

OLS 係数が経済的に解釈できるためには、説明変数と被説明変数の次元が一致していなければならない。

✅ 正しい例❌ 誤った例
EPS/BPS [円/株] → 株価 [円/株]ROE [%] → 時価総額 [百万円]
ログリターン [無次元] → ログリターン [無次元]自己資本比率 [%] → 株価 [円]

OLS実装（numpy + scipy + statsmodels）

β = (X'X)⁻¹ X'y # 正規方程式 最小二乗法: numpy.linalg.lstsq（SVD ベース、数値安定） p値計算: scipy.stats.t.sf（df < 30 でも正確） 詳細診断: statsmodels.OLS（Durbin-Watson・Jarque-Bera・F検定・HC3 SE） 返り値: beta, yhat, r2, adj_r2, rmse, mae, se, t_stat, p_value, df, rank, condition_number

係数の有意性（標準誤差・t統計量・p値）

σ² = SSE / (n - p) # 残差分散 Var(βᵢ) = σ² × [(X'X)⁻¹]ᵢᵢ # 係数の分散 SE(βᵢ) = √Var(βᵢ) # 標準誤差 tᵢ = βᵢ / SE(βᵢ) # t統計量 pᵢ = 2 × (1 - Φ(|tᵢ|)) # 両側p値（df ≥ 30）

df = n − p。p 値は scipy.stats.t.sf による正確な計算で、df < 30 の小サンプルでも信頼できる。慣例的に p < 0.05 を有意とみなす。

詳細統計診断（statsmodels）

Durbin-Watson: 残差自己相関の検定（1.5〜2.5 なら問題なし） Jarque-Bera: 残差正規性（歪度・尖度） F統計量・p値: モデル全体の有意性検定 HC3 SE: 不均一分散に対応したロバスト標準誤差

業種別 OLS の結果に diagnostics フィールドとして含まれる。

多重共線性チェック（VIF・Pearson相関）

VIFᵢ = 1 / (1 - R²ᵢ) # R²ᵢ は特徴量 i を残りで回帰したときの決定係数

慣例: VIF > 10 または |Pearson r| > 0.9 で多重共線性ありと判断。業種別OLS結果に collinearity_warnings として併記される。

Ridge 回帰（L2 正則化）

βridge = arg min ‖Xβ - y‖² + α‖β‖² # 最適 α は CV で自動選択

業種別 OLS の regularization="ridge" パラメータで切替可能。多重共線性が顕著な業種では予測安定性が向上するが、SE / t / p 値は定義されない（NaN）。実装は sklearn.linear_model.RidgeCV。

1. 総合リターン予測 Ohlsonモデル型OLS

plugins/total_return.py

Ohlson (1995) が提唱した残余利益モデルの OLS 近似。1株当たりの財務金額から理論株価を推定し、現在株価との乖離（上昇余地）と配当利回りを合算して総合リターンを予測する。

モデル定式化

# 目的変数 y = stock_price [円/株] # 説明変数（全て [円/株] — 次元整合） X = [EPS, BPS, OPS_CF, DPS] # cf_ops_ps の推計 発行株式数 ≈ bs_total_equity / bs_bps cf_ops_ps = cf_operating_cf / 発行株式数 # 前処理 X_raw → winsorize(p1–p99) → z-score正規化 → X_norm y_raw → log変換 → z-score正規化 → y_norm # OLS ŷ_norm = β₀ + β₁·EPS_norm + β₂·BPS_norm + β₃·CF_norm + β₄·DPS_norm # 逆変換（exp上限: exp(15) ≈ 3.3百万円/株） ŷ = exp( min(ŷ_norm·σ_y + μ_y, 15.0) ) [円/株] # 総合リターン 上昇余地 = (ŷ − 実際株価) / 実際株価 × 100 [%] 総合リターン = 上昇余地 + 配当利回り [%]

係数の経済的解釈

係数解釈単位
β₁ (EPS)implied P/E 倍率無次元
β₂ (BPS)implied P/B 倍率無次元
β₃ (CF)implied Price/CF 倍率無次元
β₄ (DPS)implied 配当割引倍率無次元
β_sector_i業種 i の基準業種に対する log 価格水準差log 単位

係数が経済的に意味を持つのは、説明変数と被説明変数の次元が全て [円/株] で一致しているため。

業種固定効果（オプション・デフォルト ON）

ŷ = β₀ + β₁·EPS + β₂·BPS + β₃·CF + β₄·DPS + Σⱼ βsector_j · 1{industry = j}

業種ごとの P/E・P/B 水準差を One-hot ダミー変数（最初の業種を基準としてドロップ）で吸収。サンプル数 ≥ 5 の業種のみダミー化（過学習防止）。シミュレーション検証で R² 0.83 → 0.97 に改善。

主要パラメータ

パラメータデフォルト説明
use_cfTrueCF因子（1株営業CF）を使用するか
use_sector_feTrue業種固定効果（ダミー変数）を使用するか
n_folds5横断的 k-fold 分割数
top_n20上位表示件数
LOG_PRED_CAP15.0予測値の log-space 上限（≈ 3.3百万円）
限界: 発行株式数の推計は近似値（IFRS/JGAAP 混在時に精度低下）。J-Quants の IssuedShares 取得は FUTURE_TASKS Tier 2-G として残課題。

参考文献

Ohlson, J.A. (1995). "Earnings, Book Values, and Dividends in Equity Valuation."
Contemporary Accounting Research, 11(2), 661–687.
https://doi.org/10.1111/j.1911-3846.1995.tb00461.x
Feltham, G.A. & Ohlson, J.A. (1995). "Valuation and Clean Surplus Accounting for Operating and Financial Activities."

2. 業種別OLS回帰

plugins/sector_ols.py

業種ごとに独立して OLS 回帰を実行し、Ohlson モデル拡張型で株価（円/株）を推定する。業種内でモデルを構築することで、業種間の P/E・P/B 構造差の影響を排除する。

次元整合性の構造的強制（CLAUDE.md 制約）: 目的変数は stock_price [円/株] 単一に固定、説明変数は per-share [円/株] のみ。 UI/API レベルで他の組み合わせを選べないようにすることで、係数 β を「implied 倍率」として経済的に解釈可能な状態に保つ。

モデル定式化

# 業種 s 内で独立に実行（Ohlson 拡張型） ŷₛ_norm = β₀ + Σⱼ βⱼ · Xₛⱼ_norm y = stock_price [円/株] Xⱼ = per-share 財務金額 [円/株] # 派生 per-share キー (ps_*) はランタイム計算 shares = bs_total_equity / bs_bps [株] ps_<feat> = <絶対額カラム> / shares [円/株] # 業種内でのwinsorize + z-score正規化 Xₛⱼ_norm = (Xₛⱼ_winsorized − μₛⱼ) / σₛⱼ # 乖離率（gap_ratio） gap_ratio = (ŷ − y_actual) / y_actual × 100 [%] gap_ratio > 0: 割安（市場が過小評価） gap_ratio < 0: 割高（市場が過大評価） # 予測値の DB 書き込み（互換のため百万円へ換算） predicted_market_cap = ŷ_pred / stock_price × market_cap [百万円]

説明変数（全項目 [円/株] per-share）

説明変数のキーは DB永続 per-share（pl_eps, bs_bps, dps — 公式開示値）と 派生 per-share（ps_* — 絶対額カラムを発行株数で割って実行時計算）の 2 系統。

デフォルト10項目（PL/BS/CF を網羅）:

カテゴリキー説明
PL（公式）pl_epsEPS 公式値
BS（公式）bs_bpsBPS 公式値 — Ohlson モデル中核
還元（公式）dps1株配当 公式値
PL（派生）ps_revenue売上トップライン
PL（派生）ps_gross_profit粗利・コスト構造
PL（派生）ps_operating_profit本業収益
BS（派生）ps_total_assets企業規模
BS（派生）ps_total_liabilities負債規模
CF（派生）ps_operating_cf実キャッシュ創出力
CF（派生）ps_free_cf株主還元原資

派生 per-share の全選択肢は PL/BS/CF の絶対額カラムを網羅（約30項目、ps_* プレフィックス）。

実行条件: 業種最小サンプル数（デフォルト: 10社）を満たさない業種はスキップ。各銘柄について bs_bps > 0 かつ bs_total_equity > 0 が必須（株数推計のため）。

注意:
  • bs_bps または bs_total_equity が NULL/0 の銘柄は集計対象から自動除外
  • per-share 10項目以上選択時は PL同士・BS同士の比例関係から VIF>10 が頻発する。check_collinearity の警告が出た業種では Ridge への切替を強く推奨
  • 株数推計は IFRS/JGAAP 定義差・期中増資・優先株・転換社債存在時に誤差あり（FUTURE_TASKS の J-Quants IssuedShares 取得で根本解決予定）
  • 乖離分析（gap_analysis）は本プラグインの実行後でなければ利用不可。gap_ratio の収束予測には統計的根拠がない（→ モデル 3 参照）

参考文献

Fama, E.F. & French, K.R. (1992). "The Cross-Section of Expected Stock Returns."
Journal of Finance, 47(2), 427–465.
https://doi.org/10.1111/j.1540-6261.1992.tb04398.x
Greene, W.H. (2018). Econometric Analysis (8th ed.). Pearson Education.

3. 乖離分析 AR(1) MLE + フォールバック

plugins/gap_analysis.py

業種別OLSが推定した理論値と実際の時価総額の乖離率を表示し、各銘柄の年次 gap_ratio 履歴に対して statsmodels の AR(1) モデルを MLE 推定して半減期を計算する。履歴が 8 観測未満の銘柄はヒューリスティックにフォールバック。

AR(1) MLE による半減期推定（推奨パス）

xt = c + φ xt-1 + εt, εt ~ N(0, σ²) # 平均回帰条件 0 < φ < 1 # 半減期 half_life = −ln(2) / ln(φ) [年]

フォールバック: ヒューリスティック（履歴不足時）

# 履歴 < 8 観測または AR(1) 推定失敗時のみ使用 half_life = max(6, min(24, |gap| / 2)) [ヶ月]

共通: 期待乖離率と収束スコア

# n ヶ月後の期待乖離率（指数減衰） gap_t = gap₀ × exp(−ln(2) / half_life × t) # 収束スコア（参考値、0–100スケール） conv_score₁₂ₘ = max(5, min(95, 50 + gap₀ × 0.8))
出力: 各レコードに method（"ar1" / "heuristic"）、ar1_phi、n_history を併記。レスポンス全体に n_ar1_estimated / n_heuristic_fallback のサマリを返す。

OU過程との対応

dX_t = κ(θ − X_t) dt + σ dW_t # 連続時間 OU # 離散時間 AR(1) との対応（Δt = 1 年） φ = exp(−κ Δt) κ = −ln(φ)

Physical Review, 36(5), 823–841.
https://doi.org/10.1103/PhysRev.36.823

4. 株価リターン予測 月次WF-CV

plugins/price_predictor.py

日次株価履歴（OHLCV）と年次財務指標を組み合わせ、N 日先の株価対数リターンを OLS で予測する。月次ウォークフォワード CV（ルックアヘッドバイアスなし）で評価する。

目的変数

y = log(C_{t+N} / C_t) [無次元対数リターン] N ∈ {5, 20, 60} 日（ユーザー選択）

特徴量（全て無次元 — 次元整合）

変数定義範囲
ma20_dev(C − MA20) / MA20(−1, +∞)
vol60過去60日ログリターンの標準偏差[0, +∞)
rsi14RSI(14) — 下記参照[0, 100]
atr_ratioATR(14) / C_t[0, +∞)
per, pbr, roe, equity_ratio財務比率無次元
z_op_margin, z_roe, z_cf_ratio年度別Zスコア無次元
gap_ratio業種別OLS乖離率 [%]無次元

RSI(14) の計算

changes = [C_i − C_{i-1} for i in (t−14, t)] avg_gain = mean([max(c, 0) for c in changes]) avg_loss = mean([abs(min(c, 0)) for c in changes]) RS = avg_gain / avg_loss RSI = 100 − 100 / (1 + RS)

ATR(14) の計算

TR_i = max(H_i − L_i, |H_i − C_{i-1}|, |L_i − C_{i-1}|) ATR(14) = mean(TR_i for i in (t−14, t)) atr_ratio = ATR(14) / C_t

月次ウォークフォワードCV

全月度 = ["YYYY-MM", ...] （昇順） For i = min_train_months to len(全月度)−1, step = step_months: test_month = 全月度[i] train_months = 全月度[:i] # i 未満の全月 学習: train_months の全企業 × 全スナップショット テスト: test_month のスナップショットのみ 正規化パラメータ: 学習データのみから計算 min_train_months = 18（データ不足時は 6 に緩和） step_months = 3 決算ラグ補正 = 45 日（period_end から利用可能とみなす日数）
限界: 線形 OLS のため RSI の U 字型効果等の非線形関係を捉えられない。財務データは年 1 回更新のため、月次スナップショットで同じ値が繰り返し使用される。gap_ratio が NULL の場合は財務特徴量が欠損になる。

5. 横断的Zスコア正規化

database.py — calc_zscore_normalization()

財務指標を年度別に横断的な Zスコアに変換し、異なる企業間で指標の相対的な優劣を比較可能にする。

Altman の Z スコアとは別概念。 ここでの「Zスコア」は標準化スコア（standard score）であり、倒産予測モデルである Altman (1968) の Z スコアとは無関係。

数式

# 年度 y、企業 i に対して z_field_i = (field_i − μ_y) / σ_y μ_y = mean({ field_j : field_j ≠ NULL, year(j) = y }) σ_y = stdev({ field_j : field_j ≠ NULL, year(j) = y })

正規化対象フィールド

元フィールドZスコアフィールド内容
pl_revenuez_revenue売上高
op_marginz_op_margin営業利益率
roez_roeROE
equity_ratioz_equity_ratio自己資本比率
cf_ratioz_cf_ratio営業CF/売上比
pl_epsz_epsEPS
de_ratioz_de_ratioD/Eレシオ
nc_ratioz_nc_ratioネットキャッシュ比率（モデル 8）

年度別計算の理由

異なるマクロ環境（金融緩和期・引き締め期等）の年度を混在させると比較が無意味になる。年度をまたいで計算すると、例えば低金利期の高 PER 水準が正常値として紛れ込む。

参考文献（Altman との区別）

Altman, E.I. (1968). "Financial Ratios, Discriminant Analysis and the Prediction of Corporate Bankruptcy."
Journal of Finance, 23(4), 589–609. — 本アプリの Zスコアとは別概念
https://doi.org/10.1111/j.1540-6261.1968.tb00843.x

6. Zスコア重み付けスコアリング おすすめ銘柄

plugins/recommend.py

各指標の Zスコアを重み付け線形結合してスコアを計算し、企業をランキングする。

スコア計算式

score_i = Σⱼ∈present (weight_j × z_metric_j_i) / Σⱼ∈present |weight_j| present : 企業 i において値が NULL でない指標の集合 weight_j : 指標 j の重み（ユーザー設定） z_metric_j_i: 企業 i の指標 j の Zスコア（年度別正規化済み）

weighted mean で計算するため、指標カバレッジが異なる銘柄を公平に比較できる。

カバレッジフィルタ

coverage_i = Σⱼ∈present |weight_j| / Σⱼ |weight_j|

min_coverage（デフォルト 0.5）未満の企業はランキングから除外する。1.0 指定で全指標が揃った企業のみ対象。

限界: ウェイト設定に数学的根拠はなく直感的なヒューリスティック。モデルの有効性はバックテストで検証すること。

7. バックテスト

api.py — GET /api/backtest

過去 N ヶ月前の時点で確定していた財務データを使いスコアリングし、その後の実際の株価リターンを計算することでモデルの有効性を事後的に検証する。

計算ロジック

start_date = today − months_ago × 30日 1. start_date 以前に period_end が確定しているレコードで 各社の最新年度データを取得 2. recommend.py と同じ重み付きスコアで全社をランキング 3. 上位 top_n 社について: 始値 = start_date 以降の最初の終値 終値 = 最新の終値 4. 実績リターン = (終値 − 始値) / 始値 × 100 [%] # ベンチマーク benchmark = スコアリング対象企業（最大500社）の平均リターン 超過収益 = 上位N社平均リターン − ベンチマーク平均
バイアスに注意:
• 生存バイアス: 現在 DB に存在する企業のみ対象（過去に上場廃止した企業は含まれない）
• 決算ラグ: period_end ≤ start_date の制約のみで、実際の報告書公開日（+45–60日）は未考慮
• ベンチマーク: 全上場銘柄ではなくDBに収録された企業の部分集合

8. ネットキャッシュ分析 清原達郎式

plugins/net_cash_analysis.py

清原達郎『わが投資術』(2024) で提唱された ネットキャッシュ および ネットキャッシュ比率 で割安株をスクリーニングする。OLS・回帰モデルを介さず会計値からの直接計算であるため、推定誤差が混入せず堅牢である点が特徴。

数式

# ネットキャッシュ（円） NC = 流動資産 + 投資有価証券 × 0.7 − 総負債 # ネットキャッシュ比率（無次元） nc_ratio = NC / 時価総額 # 実装上の単位調整（market_cap は百万円単位） nc_ratio = NC / (market_cap × 1_000_000)

銘柄選別基準（清原氏）

比率意味
nc_ratio ≥ 1.0現金で買える 時価総額より多くのネットキャッシュを保有
nc_ratio ≥ 0.5半額バーゲン 時価総額の半分以上をネットキャッシュで保有
nc_ratio ≤ 0負債超過 ネットキャッシュがマイナス

投資有価証券に 0.7 を乗じる理由

清原氏の経験則。投資有価証券は (1) 時価評価のブレで簿価との乖離が大きく、(2) 売却時に含み益課税（法人実効税率 ≈ 30%）が発生するため、保守的に 簿価の 70% でカウントする。実装定数 INVESTMENT_DISCOUNT = 0.7。

計算フロー

# 1. 財務収集時（calc_derived） net_cash = current_assets + investment_securities × 0.7 − total_liabilities → FinancialRecord.net_cash カラムに保存 # 2. 市場データ更新時（update_market_data_only） nc_ratio = net_cash / (market_cap × 1_000_000) → FinancialRecord.nc_ratio カラムに保存 # 3. Zスコア正規化（_calc_zscore_for_year） z_nc_ratio = (nc_ratio − μ_year) / σ_year → FinancialRecord.z_nc_ratio カラムに保存

投資有価証券の XBRL マッピング

会計基準XBRL 要素
JGAAPInvestmentSecurities / InvestmentsInSecurities / ShortTermInvestmentSecurities
IFRSOtherFinancialAssetsNonCurrentIFRS（近似値）

IFRS には完全に対応する科目がないため「非流動その他金融資産」で近似する。古いレコード（本機能リリース前）は bs_investment_securities = NULL で、内部計算では 0 として扱う（簡易 NCAV (Graham 1934) 相当の値を返す）。

限界:
• 0.7 倍ルールは経験則で、個別銘柄の含み益・含み損・政策保有株は反映しない
• 特別損失リスク: 売掛金・棚卸資産が現金化できる保証はない
• IFRS 採用企業では OtherFinancialAssets の区分が異なる場合がある
• 商社・金融・REIT 等は BS 構造が特殊で指標が機能しない業種がある（業種フィルタで除外推奨）
• 古いレコードは投資有価証券未収集（NULL=0扱い）のため過小評価方向のバイアスあり
