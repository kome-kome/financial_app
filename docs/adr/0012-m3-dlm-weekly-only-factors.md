# M-3（時変マクロβ DLM）は週次高頻度ファクター専用とし、月次以下のマクロ系列を組み込まない

## Status

accepted（2026-07-12）。Issue #310（M-3 への月次以下マクロ系列組み込みの方針検討）の設計決定。by-design で #310 をクローズする。

## Context

M-1（`macro_risk_return.py`）・M-2（`macro_gbdt.py`）は `plugins/macro_snapshots.py` の `MACRO_FEATURE_OPTIONS`（31系列）を共用し、日本マクロ統計（実質GDP・失業率・貿易収支・鉱工業生産/在庫指数・CPIコア・日銀短観4種・M2・CGPI・マネタリーベース・OECD CLI・IMF WEO見通し）や社債スプレッド・BEI を特徴量として選択できる。

一方 M-3（`macro_dlm.py`）は独自の `_DLM_MACRO_MAP`（13系列）を持ち、FX・株価指数・商品・金利の**週次系列のみ**で、上記の月次〜四半期〜年次系列を持たない。Issue #310 は「意図的除外を示す ADR が無く、未検討のままスコープ外になっている可能性が高い」として方針の明文化を求めた。

### 技術的な核心（なぜ低頻度系列が M-3 に馴染まないか）

M-3 の観測モデルは **「銘柄の週次対数リターン $y_t$ ~ $[1,\ \Delta\text{factor}_{1,t},\ \Delta\text{factor}_{2,t},\ \dots]$」**（`_build_series`）。説明変数は各ファクターの**週次変化**である：

- `kind="logret"`：$\log(\text{level}_t / \text{level}_{t-1})$（指数・FX・商品）
- `kind="diff"`：$\text{level}_t - \text{level}_{t-1}$（金利）

マクロ水準は週次グリッドへ forward-fill される（`_macro_change_builder` の `level_at`）。ここで**月次以下の系列は月内で定数**になるため、その週次変化は「月に1度のスパイク＋残り全週ゼロ」というほぼゼロの疎な列になる。IMF WEO のような年次系列に至っては年内ほぼ定数＝1年に1度しか動かない。時変係数を逐次ベイズ更新する DLM に対し、大半の週で説明変数が 0 の列は係数をほぼ更新できず、情報量が構造的に乏しい。

この弱点は**既に M-3 内で実証・明記済み**：唯一の月次系列 `dlm_jp10y`（`JP10Y_FRED`・月次）に「週次差分は多くの週でゼロ＝情報量に限界あり（日次ソース確保は将来課題）」とコメントがある。これは日次の日本10年金利（Yahoo `^JGB`）が廃止（404）で取得できないための**やむを得ない例外**であり、月次系列を積極採用した先例ではない。

さらに `_MIN_FACTOR_COVERAGE`（薄いファクターの自動除外・既定0.5）は**この問題を検出できない**：forward-fill 後は全週に値が入りカバレッジ ≈ 1.0 になるため、情報量の乏しい月次系列は除外されずモデルに残り、弱い/ノイジーな列として推定を薄める。

### 役割分担（既にカバー済み）

月次以下のマクロは M-1/M-2 が**スナップショット方式**（level/yoy/zscore の水準値を特徴量に投入）で扱っており、低頻度データに適合している。M-3 の固有価値は「高頻度ファクターに対する**時変**感応度 β の推定」であり、両者は補完関係にある。低頻度系列を M-3 に足すことは、この固有価値を薄めるだけで新情報を生まない。

## Decision

1. **M-3 は週次高頻度ファクター専用とする。** 月次・四半期・年次のマクロ系列（`macro_snapshots._MACRO_MAP` の JP 実体経済・物価・マネー・サーベイ・OECD CLI・IMF WEO 等）を `_DLM_MACRO_MAP` に**追加しない**。

2. **`dlm_jp10y`（月次 `JP10Y_FRED`）は grandfathered な例外として現状維持。** 日次の日本10年金利ソースが確保できない代替であり、既存コメントの限界注記も残す。日次ソースが将来入手できれば週次系列へ差し替える（このときこそ M-3 の設計に合致する）。

3. `_DLM_MACRO_MAP` 冒頭に本 ADR を参照する設計コメントを置き、将来「M-1/M-2 と同じ月次系列を M-3 にも足すべきでは」と再検討されたときに、観測モデル（週次変化回帰）との不整合という理由へ即座に辿れるようにする。

4. Issue #310 は **by-design（wontfix）でクローズ**する。

## Considered Options

- **厳選した月次系列を `diff` で opt-in 追加**（却下）：CPIコア・鉱工業生産・失業率などを `dlm_jp10y` と同様に足す案。実装は自明（`_DLM_MACRO_MAP` に数行）だが、週次変化が大半ゼロで情報量が乏しく、`_MIN_FACTOR_COVERAGE` の自動除外も効かないため弱い列がモデルに常駐する。ユーザーが選べても誤誘導になりうる。M-1/M-2 で同系列を選ぶ方が適切。
- **混合頻度 DLM へ再設計**（却下・priority:low に不釣り合い）：月次系列を「週次変化」ではなく「z スコア水準」の緩やかな回帰子として観測式に混在させる案。理論的には可能だが、`dlm_filter` の観測モデル（現状は全列が週次変化）を作り替える大工事で、時変 β の解釈も水準回帰子とで二層化する。#310 の優先度（low）に見合わない。将来 M-3 を「高頻度反応 β ＋ 低頻度レジーム水準」の2層に拡張する構想が出たら、その時点で新規 ADR として検討する。

## Consequences

- **実装変更なし（ドキュメント＋コメントのみ）**：`_DLM_MACRO_MAP` は13系列のまま。既存の recovery/calibration/smoke テスト（`tests/test_macro_dlm.py`）・producer 永続化・sell_ranking 連携・モデル比較（OOF）は不変。
- **設計意図の明文化**：以後 M-3 に月次系列を足す提案は本 ADR で退けられる（または本 ADR を supersede する新 ADR を要する）。Issue #310 のような「未検討で放置」の再発を防ぐ。
- **将来の余地**：日次の日本10年金利ソース確保時は `dlm_jp10y` を週次系列へ差し替え。M-3 の2層（高頻度反応＋低頻度レジーム）拡張は別 ADR で扱う。
