## 0. この資料の歩き方

M-1 は、このプロジェクトで**最も長く議論された**分析モデルです。一見すると専門用語の塊ですが、
やりたいことは一言でいうと次の通りです。

> **「いまの経済環境（マクロ）を踏まえて、各銘柄の『期待リターン』と『リスク』を見積もり、
> リスクのわりにリターンが良い銘柄を、一枚の散布図の上で選び出す」**

下の図が完成イメージです。点ひとつが1社で、**上に行くほど儲かりそう**・**右に行くほど危険**。
左上のフチ（太線）に並ぶのが「おすすめ候補」です。

```
 期待リターン μ（高いほど良い ↑）
  高 │                          ● E社
     │                  ● C社        ● F社
     │          ● B社                     ● G社
     │   ● A社            ● D社
     │ ╱（パレートフロンティア＝左上のフチ）
  低 │╱___________________________________→ リスク（右ほど危険）
       低                                高
```

本書は、この一枚の図を**完全に読めるようになる**ことをゴールに、次の順で説明します。

| 章 | 問い | キーワード |
|---|---|---|

| 3 | なぜ企業ごとに反応が違うの？ | **交差項**（このモデルの心臓部） |
| 4 | 「リスク」をどう数値にするの？ | ボラティリティ・予測の不確実性 |

## 1. そもそも何を作りたいのか

### 1-1. 既存モデルとの違い

このアプリには既に「おすすめ銘柄」や「バリュエーション分析」などの分析モデルがあります。それらは

> 「**この企業**の財務を見ると、来年の株価はこれくらいになりそう」

と、**1社ずつの予測**を出すものでした。M-1 が新しく持ち込む発想は2つです。

1. **経済全体の風向き（マクロ）を予測に織り込む**
   - 同じ財務内容の会社でも、「円安の年」と「円高の年」では株価の動き方が違う。その違いを取り込む。
2. **「当たりそうか（リスク）」も同時に見積もる**
   - 予測リターンが高くても、その予測が**当てにならない**なら手を出しにくい。「リターン」と「自信のなさ＝リスク」をセットで評価する。

### 1-2. 最終アウトプット

ユーザーが画面で受け取るのは、**冒頭のリスク-リターン散布図**と、**おすすめ銘柄ランキング表**です。
スライダーで「どれくらいリスクを嫌うか」を調整すると、おすすめの顔ぶれが変わります。

## 2. 「マクロ要因」とは ― 経済全体の天気予報

**マクロ要因**とは、特定の1社ではなく**経済全体に効く大きな変数**のことです。
個別企業の話（ミクロ）に対して、全体の話（マクロ）。天気にたとえると、
「個社の財務」が服装なら、「マクロ」はその日の天気そのものです。

このアプリが収集している主なマクロ要因（`macro_data` テーブル）：

| マクロ要因 | 何の数字？ | 上がると…（ざっくり） |
|---|---|---|
| **USDJPY（ドル円）** | 1ドルが何円か。例: 150円 | 数字が上がる＝**円安**。輸出企業に追い風 |
| **US10Y（米10年金利）** | アメリカ国債の利回り | 上がると世界の資金が米国へ。ハイテク株などに逆風になりやすい |
| **JP10Y（日本10年金利）** | 日本国債の利回り | 上がると銀行は喜び、借金の多い企業は苦しい |
| **SP500（米国株指数）** | アメリカ株500社の平均 | 世界の投資家心理の温度計。上がると日本株も上がりやすい |

### 2-1. 大事な約束①：「水準」ではなく「変化・サプライズ」を使う

ここが地味だけど重要なポイントです。M-1 は「ドル円が**150円**」という**水準（レベル）**は説明変数に使いません。
代わりに「ドル円が**1年で +20%**動いた」という**変化（サプライズ）**を使います。

なぜか？ 株価は「織り込み済みの情報」には反応しないからです。みんなが知っている「150円」という事実は
すでに株価に入っています。株価を動かすのは**予想外の変化**の方。
これは学術的にも APT（裁定価格理論, Chen-Roll-Ross 1986）の教えで、
「マクロは*水準でなく innovation（変化分）*で価格付けされる」とされています。

そこで M-1 は2種類の「無次元（単位のない比率・スコア）」に変換します。

| 変換 | 対象 | 計算式 | 意味 |
|---|---|---|---|
| **YoY 変化率** | USDJPY, SP500 | (今の30日平均 − 1年前の30日平均) ÷ 1年前の30日平均 | 「1年で何%動いたか」 |
| **5年 Z スコア** | US10Y, JP10Y | (今の30日平均 − 過去5年平均) ÷ 過去5年の標準偏差 | 「過去5年の中で今は何σ高い／低いか」 |

> **Z スコアとは？** 「平均からどれくらい離れているか」を、ばらつきの大きさ（標準偏差σ）を物差しにして
> 測った値です。Z=0 なら平均ちょうど、Z=+2 なら「かなり高め（平均＋2σ）」。
> 単位（%や円）が消えるので、桁の違う指標どうしを公平に比べられます。

## 3. このモデルの心臓部 ―「同じニュースでも企業ごとに反応が違う」

ここが M-1 でいちばん大切な発想です。**ゆっくり読んでください。**

### 3-1. 直感：円安は誰の味方か

「円安（ドル円が上がる）」というニュースを考えます。これは全企業に**同じように**効くでしょうか？
いいえ、**真逆**に効きます。

- 🚗 **トヨタ（輸出企業）**：海外で稼いだドルを円に換えると金額が増える → **株価にプラス**
- ⚡ **電力会社（輸入依存）**：燃料を高いドルで買わされる → コスト増 → **株価にマイナス**

つまり「円安」という**たった1つのマクロ事象**でも、

> **企業の性質（セクター・財務構造）によって、効き方の符号も大きさも変わる。**

この「企業ごとに異なる効き方の強さ」を、専門用語で **β（ベータ＝感応度）** と呼びます。
トヨタの円安βはプラス、電力会社の円安βはマイナス、というわけです。

### 3-2. それを数式に入れる方法 ＝「交差項」

では「企業ごとに違うβ」をどうやって回帰モデル（OLS）に教えるのでしょう。
答えが **交差項（こうさこう, interaction term）**＝**2つの変数の掛け算**です。

ふつうの回帰はこう書きます（足し算だけ）：

```
リターン = a×(円安) + b×(輸出企業ダミー) + …
```

これだと「円安は全社に一律 a だけ効く」としか言えません。そこで掛け算の項を足します：

```
リターン = a×(円安) + b×(輸出企業ダミー)
         + c×(円安 × 輸出企業ダミー)   ← これが交差項
         + …
```

> **ダミー変数とは？** 「あてはまるなら1、そうでないなら0」のスイッチ変数。
> 「輸出企業ダミー」は、トヨタなら1、電力会社なら0。

この **`c×(円安 × 輸出企業ダミー)`** が魔法です。
- トヨタ（ダミー=1）には `a + c` の効きになり、
- 電力会社（ダミー=0）には `a` だけの効きになる。

c がプラスに推定されれば「輸出企業は円安に**追加で**強く反応する」とデータが言っている、ということ。
**1本の式の中で、企業タイプごとに別々の感応度**を持たせられるのです。これが APT のいう「異質な感応度」の正体です。

### 3-3. M-1 が用意する交差項

M-1 は財務×マクロの掛け算を用意します。

| 交差項 | 例 | 何を捉える？ |
|---|---|---|
| **財務 × マクロ** | （自己資本比率）×（金利変化） | 「借金が多い会社は金利上昇に弱い」など、**財務構造ごとの感応度** |

### 3-4. 説明変数は全部「無次元」に揃える（重要な制約）

このプロジェクトには鉄の掟があります（CLAUDE.md「次元整合性」）。
**「説明変数と被説明変数は同じ次元（単位）に揃える」** というものです。

- 予測したいもの（被説明変数）μ ＝「1年先リターン・年率」＝ **無次元**（％は単位のない比率）。
- だから説明変数も**全部無次元**にする。「円」や「百万円」のような単位を持つ生の金額は使わない。

M-1 が使う説明変数（すべて無次元）：

- **財務比率**：PER・PBR・ROE・自己資本比率・R&D集約度・減価償却集約度 ＋ それらの年度別Zスコア
- **モメンタム**：12-1ヶ月リターン（「1年前〜1ヶ月前の値上がり率」。直近1ヶ月を除くのが定番）
- **マクロ**：第2章の YoY 変化率・5年Zスコア
- **交差項**：上記どうしの掛け算

> **モメンタムとは？** 「最近上がっている株は、もう少し上がり続けやすい」という経験則（Jegadeesh-Titman 1993）。
> 単独で最も強いアノマリー（市場の謎の規則性）の1つとして知られ、無料で手に入る強力な説明変数です。

## 4. 「リスク」を数値にする ― 予測のブレ幅

ユーザーの当初の要望はこうでした：

> 「マクロ×説明変数の相関で**予測可能性の大小**が決まる。予測可能性を**予測値の分散（リスク）**として定義したい」

噛み砕くと――「ある銘柄は予測が当てやすく、別の銘柄は当てにくい。当てにくさ＝リスクとして扱いたい」。
これは正しい直感です。M-1 ではリスクを**3つの角度**から測ります（詳細は第7章）。いったんここでは、
いちばん基本の **R2＝実現ボラティリティ** だけ押さえます。

### 4-1. R2：実現ボラティリティ＝「株価のジェットコースター度」

**ボラティリティ**とは、株価の**上下の激しさ**のこと。
毎日コツコツ動く株は低ボラ、乱高下する株は高ボラ。

M-1 の R2 は「予測基準日の**直前1年間**の週次リターンの標準偏差 ×√52」で測ります。

- **標準偏差**＝ばらつきの大きさ。大きいほど値動きが荒い。
- **×√52**＝週次のばらつきを年率に換算する“お作法”（1年は52週なので）。
- ポイント：**過去のデータだけ**で計算する＝未来を覗き見しない（**リークなし**。これは予測の公正さに必須）。

R2 が大きい＝過去よく暴れた＝リスクが高い、という素直な指標です。これが散布図の**横軸（既定）**になります。

## 5. リターンとリスクを1枚の図に ― 効率的フロンティア

### 5-1. Markowitz の発明

1952年、ハリー・マルコウィッツという人が「**投資はリターンとリスクの2軸で考えるべき**」という、
いまでは当たり前すぎる発想を初めて数学にしました（ノーベル賞）。彼の描いた図がまさに冒頭の散布図です。

- **縦軸**：期待リターン（高いほど良い）
- **横軸**：リスク（低いほど良い）

各銘柄を点として置くと、雲のように散らばります。その雲の**左上のフチ**に注目してください。

### 5-2. パレートフロンティア（効率的フロンティア）

```
 リターン ↑
   │              ★E                ← 左上のフチ（太線）に並ぶ★が
   │         ★C       ○F              「効率的」な銘柄
   │     ★B      ○D
   │  ★A   ○         ○G            ○ は「誰かに負けている」銘柄
   │ ╱（フロンティア）                  （同じリスクでもっと上がいる）
   └────────────────────→ リスク
```

★ の銘柄たちは、こういう意味で**誰にも負けていません**：

> 「**同じリスクなら、これ以上リターンが高い銘柄は存在しない**」
> 「**同じリターンなら、これ以上リスクが低い銘柄は存在しない**」

この“負けていない”集合を **パレート最適**、それが描く左上の境界線を
**効率的フロンティア（パレートフロンティア）** と呼びます。M-1 はこの ★ を**色分けで強調**します。
○（フロンティアの内側にいる、誰かに見劣りする銘柄）は、わざわざ選ぶ理由がありません。

## 6. なぜ「リターン軸」が危ないのか ― 収縮という安全装置

ここは M-1 の設計でいちばん「大人の知恵」が効いている部分です。

### 6-1. 落とし穴：予測リターンをそのまま信じると失敗する

「期待リターンが高い銘柄を上から選べばいい」――素朴にはそう思えます。**でもこれが罠なのです。**

予測には必ず**誤差**が混じります。誤差にはプラス方向もマイナス方向もある。
すると、ランキング上位に来るのは「本当に良い銘柄」とは限らず、
**「誤差でたまたま高く出てしまった銘柄」**が紛れ込みます。上位だけ選ぶと、
この“まぐれ当たり”ばかり拾ってしまう。これを **誤差最大化（error maximization）** と呼びます
（Michaud 1989 の有名な指摘）。

さらに残酷な研究結果があります。Chopra-Ziemba (1993) によれば、

> **期待リターンの推定ミスは、リスク推定のミスより約11倍も有害。**

つまり「**危険なのはリターン軸の方**」。M-1 はこの教訓を設計の中心に据えています。

### 6-2. 対策：収縮（shrinkage）＝平均へ引き戻す

対策はシンプルかつ強力です。各銘柄の予測リターンを、**そのセクターの平均**の方へ少し引き戻す。
これを **収縮（しゅうしゅく, shrinkage）** といいます（James-Stein 1961, Black-Litterman 1992）。

```
   生の予測（暴れん坊）        収縮後（落ち着いた）
   A社: +30% ─┐               A社: +18%
   B社: +25%   ├─ セクター平均 +10% ←へ引き戻す
   C社:  -5% ─┘               C社:  +2%
```

- **引き戻す強さは銘柄ごとに変える**のがミソ。
- **予測の自信がない銘柄ほど、強く平均へ引き戻す。** 自信のあてにはリスク指標 **R1**（次章）を使う。

直感的には「よく知らないことほど、とりあえず無難な平均に寄せておく」という、人間の慎重さと同じです。
ただし、実際にバブルチャートの縦軸に使うのは**生の予測 μ_raw** です。セクター平均への収縮はシグナルが弱い局面では全銘柄を同じ値に潰してしまうため、可視化・効用計算には不適と判断し廃止しました。

## 7. リスクの3兄弟 ― R1・R2・R3 の役割分担

ユーザーから「リスクを3つ計算して、その中の2軸を選んで3D散布図にできないか？」という案も出ました。
検討の結果、**3つを対等な3軸にするのではなく、役割を分ける**設計に落ち着きました。
3つはそれぞれ「測っているものが違う」からです。

| 名前 | 何を測る？ | たとえると | 図での役割 |
|---|---|---|---|
| **R2** 実現ボラ | 株価そのものの**値動きの荒さ**（過去1年） | ジェットコースターの激しさ | **横軸（既定）** |
| **R1** 予測不確実性 | モデルが**この1社をどれだけ自信もって予測できたか** | テストで自信のない設問 | **点の大きさ・色**（＋収縮の重み） |
| **R3** モデル信頼性 | この**企業タイプ全体**をモデルがどれだけ説明できたか | この単元の通信簿 | **補助**（バケット単位） |

### 7-1. R1：予測不確実性（この1点をどれだけ信じるか）

回帰モデルは、**学習データによく似た企業**は得意に予測できますが、
**学習データから遠い（珍しい特徴の）企業**は自信が持てません。
この「自信のなさ」を統計的に測ったのが R1（OLS の予測分散 `se_obs`）です。

R1 は2役を兼ねます：
1. 散布図で**点の大きさ・濃さ**として表示（大きく濃い＝高信頼）。
2. 第6章の**収縮の強さ**を決める（R1 が大きい＝自信ない＝強く平均へ引き戻す）。

### 7-2. R2：実現ボラ（第4章のおさらい）

過去の値動きの荒さ。**個社ごとに厳密**に出せる、いちばん素直なリスク。Sharpe レシオ（後述）の分母にもなります。

### 7-3. R3：モデル信頼性（このタイプをモデルは得意か）

R1 が「**1社の**予測のあやふやさ」なのに対し、R3 は
「**このセクター×企業規模のグループ全体**について、モデルが過去どれだけ当ててきたか」の成績表です。
グループ（バケット）単位で、検証時の予測誤差（RMSE）を見ます。

> **なぜ個社でなくグループ？** 日本企業は1社あたり年1回しか決算が出ないので、
> 1社だけだと検証サンプルが数個しかなく不安定。似た者どうしをまとめて初めて信頼できる成績が出ます。
> R1（イン・サンプルの偏り）と R3（アウト・オブ・サンプルのグループ誤差）は、
> 「モデルの信頼性」を内と外の両面から見たもの。R2 だけは別物（純粋な価格リスク）です。

## 8. どうやっておすすめを選ぶか ― 効用関数とパレート

散布図が描けたら、最後に**おすすめランキング**を作ります。方法は2つを併用します。

### 8-1. 効用関数 U ＝ リターンとリスクを1つの点数に合体

人によって「リスク嫌いの度合い」は違います。それを反映する1本の点数式が **効用関数（こうようかんすう）**：

```
   U = μ_raw  −  λ × R_axis
       (期待リターン)  (リスク回避度)(リスク)
```

- **λ（ラムダ）** はユーザーが**スライダー**で動かす「リスク嫌い度」。
- **λ = 0**：リスクを気にしない → リターン μ が高い順におすすめ。
- **λ 大**：リスクを強く嫌う → リスク R2 が低い銘柄を優先。

U が高い順に並べれば、その人の好みに合ったランキングになります。
λ を動かすとランキングが組み替わる――この**対話性**が M-1 の使い心地の核です。

> **Sharpe レシオとの関係**：実務では「リターン ÷ リスク」（Sharpe レシオ, Sharpe 1966）で効率を測ります。
> 生リターンの大きさだけで競わせない、という第6章と同じ精神です。U はその引き算版だと思ってください。

### 8-2. パレートフロンティア強調（第5章の実用化）

効用ランキングに加えて、第5章の **★（パレート最適銘柄）を色で強調**します。
λ をどう設定しようと、★ の銘柄は「誰かに明確に見劣りする」ことがない安全な候補。
「まずフロンティア上から見てみる」という探し方ができます。

## 9. 「低リスクほど得」という逆説 ― 低ボラ・アノマリー

最後に、結果の**見え方**について心の準備を。

普通は「ハイリスク・ハイリターン（リスクを取るほど儲かる）」と教わります。
だから散布図も**右肩上がり**（右に行くほど上がる）になりそうな気がします。**ところが現実は違う**のです。

数々の実証研究が、

> **「低リスクの株の方が、リスク調整後はむしろ報われやすい」**

という逆説を見つけています。これを **低ボラティリティ・アノマリー**、
あるいは **Betting-Against-Beta**（Frazzini-Pedersen 2014）と呼びます。

その結果、M-1 の散布図は右肩上がりにならず、
**左上（低リスク・高リターン）に良い銘柄のかたまり**ができる、と想定しています。
これはバグではなく**むしろ正しい姿**。M-1 はこれを無理に整形せず、ありのまま提示します。
「フロンティアは非単調（きれいな右肩上がりではない）」を前提に図を描く、という設計です。

## 10. 全部つなげる ― 一気通貫の具体例

ここまでの部品を、1社「A社（輸出メーカー）」を例に最初から最後まで通してみます。

```
【入力をそろえる】
  ① 財務比率を計算        … PER=12, PBR=1.1, ROE=9%, 自己資本比率=55% …（無次元）
  ② モメンタムを計算       … 12-1ヶ月リターン=+8%
  ③ マクロを変化に変換     … ドル円 YoY=+18%（円安）, 日本金利 5年Z=+1.2
  ④ 交差項を作る          … (輸出ダミー=1)×(円安+18%), (自己資本比率)×(金利Z) …

【モデルを組む（LassoLarsIC BIC）】
  ⑤ 交差項の候補から LassoLarsIC(BIC) で有効な特徴量を一括選択。
     L1 正則化が共線性を内在的に処理するため VIF 監視は不要。
     → 最終的に 15〜25 項のスッキリした式に収束。

【リターンとリスクを出す】
  ⑥ その式でA社の生の予測リターン μ_raw を算出  … +22%
  ⑦ R2（過去1年の実現ボラ）を算出              … 19%

【図に置く・点数化する】
  ⑧ 散布図に配置：縦=μ_raw +22%, 横=19%
  ⑨ 効用 U = 22% − λ×19% を計算（λはスライダー）
  ⑩ A社がパレートフロンティア上なら★で強調

【ユーザーの操作】
  ⑪ λスライダーを動かすとランキングが組み替わる。
     左上の★クラスタから、好みのリスク水準の銘柄を選ぶ。
```

これが M-1 の全体像です。**「マクロを変化で入れる → 交差項で企業ごとの感応度を捉える →
μ_raw をリターン軸・リスクと並べて図にする → 効用とパレートで選ぶ」**。
この4ステップを覚えれば、もう冒頭の散布図は完全に読めます。

## 11. 偉大な投資家・研究から学んだこと（設計の裏付け）

M-1 の設計は思いつきではなく、過去の名だたる研究・投資手法を調査して裏付けを取りました。
要点だけまとめます。

| 出典 | 教訓 | M-1 への反映 |
|---|---|---|
| **Markowitz 1952 / Sharpe** | 投資はリスクとリターンの2軸。効率はリターン÷リスクで測る | 散布図そのもの。生リターンでなく効用/Sharpe でランク |
| **Fama-French 3/5因子** | 割安(PBR)・収益性(ROE)・投資は実証されたファクター | 財務比率を説明変数に採用。係数の符号を常識でサニティチェック |
| **APT / Chen-Roll-Ross 1986** | 金利・為替は理論的に価格付けされる。感応度は企業ごとに違う | **交差項そのもの**。マクロは水準でなく変化で入れる |
| **Black-Litterman 1992 / James-Stein** | 生の予測リターンは誤差最大化を招く | **μ をセクター平均へ収縮**、重みは予測信頼度 R1 |
| **低ボラ・アノマリー / Frazzini-Pedersen 2014** | 低リスク株がむしろ報われる | フロンティアは非単調と想定し、整形せず提示 |
| **Michaud 1989 / Chopra-Ziemba 1993 / DeMiguel 2009** | 危険なのはリターン軸（誤差は分散の約11倍有害）。凝りすぎは1/Nに負ける | 収縮を最重視。推奨ルールはシンプルに保つ |
| **Jegadeesh-Titman 1993** | モメンタムは単独最強級のアノマリー | 12-1ヶ月モメンタムを説明変数に追加 |

**ひとことでの結論**：M-1 の骨格（交差項でマクロ感応度を捉え、リスク-リターン平面で選ぶ）は学術的に妥当。
そのうえで「収縮」「Sharpe的ランク」「マクロは変化で」「モメンタム追加」「時系列CV」を足すことで、
**実運用で陥りがちな落とし穴（特にリターン軸の過信）を避ける**設計になっています。

## 12. 用語集

| 用語 | やさしい説明 |
|---|---|
| **マクロ要因** | 経済全体に効く大きな変数（為替・金利・株価指数）。個社の話＝ミクロの対義語 |
| **無次元** | 単位（円・%）を持たない数。比率やスコア。次元を揃えるために重要 |
| **YoY** | Year over Year。「1年前と比べて何%」 |
| **Z スコア** | 平均からの距離を、ばらつき（標準偏差σ）を物差しに測った値。Z=+2 は「平均＋2σ」 |
| **β（ベータ）／感応度** | あるマクロ変化に対して、株価がどれだけ反応するかの強さ |
| **交差項** | 2つの変数の掛け算項。「企業ごとに違う感応度」を1本の式に入れる仕掛け |
| **ダミー変数** | 「あてはまれば1、なければ0」のスイッチ変数 |
| **OLS（最小二乗法）** | いちばん基本の回帰。誤差の二乗和が最小になるよう直線を引く |
| **ボラティリティ** | 値動きの荒さ。標準偏差で測る。高いほどジェットコースター |
| **モメンタム** | 「最近上がった株は上がり続けやすい」という経験則 |
| **効率的フロンティア／パレート最適** | 「誰にも見劣りしない」銘柄が並ぶ、散布図の左上のフチ |
| **収縮（shrinkage）** | 自信のない予測を、無難な平均の方へ引き戻す安全装置 |
| **効用関数 U** | リターンとリスクを「リスク嫌い度λ」で1つの点数にまとめる式 |
| **Sharpe レシオ** | リターン÷リスク。リスク1単位あたりの儲けの効率 |
| **低ボラ・アノマリー** | 低リスク株の方がリスク調整後むしろ報われる、という実証された逆説 |
| **VIF** | 変数どうしが似すぎていないか（多重共線性）を測る指標 |
| **walk-forward CV** | 時間順を守った検証。未来を覗き見しない正しい汎化チェック |
